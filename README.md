# TALA — نظام القيود المحاسبية الذكي (Flask)

نظام محاسبة آلي يحوّل كشوف الحسابات البنكية (الأهلي، الإنماء، الرياض) إلى قيود محاسبية جاهزة للاستيراد في QuickBooks.

## التشغيل محلياً

```bash
pip install -r requirements.txt
python app.py
```

افتح المتصفح على: http://localhost:5000

## الملفات

- `app.py` — Flask backend (كل API endpoints)
- `templates/index.html` — الواجهة الكاملة (HTML/CSS/JS)
- `matcher.py` — محرك المطابقة (Exact / Fuzzy / Rules)
- `parser.py` — تحليل كشوف البنوك (AlAhli / Alinma / Riyad)
- `exporter.py` — تصدير CSV (QuickBooks) و Excel
- `tala_data.db` — قاعدة البيانات (SQLite) — تحتوي بيانات الشركة الحقيقية

## النشر

### Railway / Render
1. ادفع المشروع لمستودع GitHub
2. اربط المستودع بـ Railway أو Render
3. Build command: `pip install -r requirements.txt`
4. Start command: `gunicorn app:app --bind 0.0.0.0:$PORT` (موجود في `Procfile`)
5. أضف Volume أو متغير `DB_PATH` للحفاظ على قاعدة البيانات بين عمليات إعادة النشر

### متغيرات البيئة
- `DB_PATH` (اختياري) — مسار قاعدة البيانات، الافتراضي `tala_data.db`
- `PORT` — يُحدَّد تلقائياً من المنصة

## شاشات النظام

1. **لوحة التحكم** — إحصائيات سريعة وآخر القيود
2. **شجرة الحسابات** — إضافة/حذف/بحث الحسابات
3. **الموردين / العملاء / المصروفات / المواقع** — بيانات المطابقة
4. **البيانات الموحدة** — عرض كل قواعد المطابقة في مكان واحد
5. **رفع كشف البنك** — رفع Excel/CSV ومعالجته تلقائياً
6. **مراجعة القيود** — اعتماد القيود المقترحة حسب نسبة الثقة
7. **التصدير** — تصدير CSV لـ QuickBooks أو Excel مفصّل أو الحفظ في قاعدة البيانات
8. **سجل القيود** — عرض/حذف القيود المحفوظة
